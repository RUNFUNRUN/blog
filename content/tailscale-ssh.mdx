# SSH接続する

実際にMacからArch LinuxにSSH接続してみます。

方法は簡単で、sshコマンドの後に表示されているIPアドレスを指定するだけです。

<ImageZoom
  src='https://i.gyazo.com/1e99dc3215c7e4b909dcb99a1b940a8b.png'
  width={1225}
  height={910}
  alt='image by gyazo'
/>

非常に高速に動作します。

Cloudflare TunnelでSSH接続したときは遅延が酷く、Neovimが使い物にならなかったのですが、
Tailscaleは快適に使えそうです。

Cloudflareのように、ブラウザからの接続もBetaで実装されているようです。

<ImageZoom
  src='https://i.gyazo.com/59ff077beb86e40cf3ed5d758a9167da.png'
  width={1378}
  height={653}
  alt='image by gyazo'
/>

...からSSH to machine...を押してユーザーを入力するとできます。

Windowsだと別windowで開きます。Macだとそのまま新規タブで開きました。

<ImageZoom
  src='https://i.gyazo.com/4f27283fab2e98ff807c1e43218fb6e7.png'
  width={789}
  height={501}
  alt='image by gyazo'
/>

ブラウザから接続すると、Machinesに追加されます。

<ImageZoom
  src='https://i.gyazo.com/98e013bc2b1fa770a22dc9f0ded2beaa.png'
  width={1378}
  height={653}
  alt='image by gyazo'
/>

このブラウザSSH接続はNeovimが正常に描画されませんでした。
CloudflareのブラウザSSHは正常に描画されていたので、改善の余地ありですね。

Mac側の`.ssh/config`には以下を設定しました。

```
Host arch
  HostName 100.69.69.94
  ServerAliveInterval 60
```

ポートフォワーディングもできました。

```bash
ssh -L 3000:localhost:3000 arch
```
